# a-share-sanity-check（A股基本面快速体检 / 红旗识别 v2）

> 目标：对用户给定的股票列表做“快速体检”，优先识别**明显的财务/治理/持续经营红旗**，并给出需要进一步核验的清单。
>
> 重要声明：本技能仅做风险识别与信息整理，不构成投资建议。

---

## 何时使用
- 用户给了一个候选池（例如 10~50 只），希望：
  - “帮我看看哪些有雷/哪些财务更稳”
  - “标出高风险：亏损、现金流差、商誉高、质押高、非经常性损益依赖”
  - “做一个投前尽调的快速版 checklist”

不适用：
- 需要完整建模估值（DCF/剩余收益等）
- 需要审计级别的财务重述或舞弊识别（只能提示风险线索）

---

## 输入
- 股票列表：`codes=[...]`（6 位代码）
- 可选：行业（金融/地产/强周期需不同阈值）
- 可选：财报期（默认最近一期 + 最近 3~5 年摘要）

**最小可运行模式（MVP）：**
- 允许用户提供一个 CSV/JSON，包含关键字段，本技能对其做规则判定输出红旗。
- 若关键字段缺失：在输出里必须标注 **`[未核验]`**（至少在“下一步核验清单”里列出缺失项）。

---

## 输出（Telegram 友好）
对每只股票输出：
- 结论标签：`相对健康/待核验/高风险（红旗≥N）`
- 红旗列表（按严重程度排序）
- 关键数值快照（便于复核口径）

并提供：
- 全局统计：红旗最多的前 N 只、常见红旗分布
- 风险提示（不构成投资建议）
- 下一步核验清单（公告/财报/审计意见/监管问询等）

## 红旗规则（v0.2，扩展版）
> 命中不等于一定有问题：仅提示需要优先核验的方向。
> 阈值需按行业校准；金融/地产/强周期与制造消费不同。

### A. 周期下行 × 高杠杆（组合红旗）
- 若（周期下行信号）且（杠杆偏高/短债压力）→ **红旗：周期下行 + 高杠杆放大器**
  - 周期下行信号（任一）：毛利率/净利率显著下行、利润下滑、价格/需求下行（可来自年报/季报/行业数据）
  - 杠杆信号（任一）：资产负债率高、有息负债高、利息覆盖弱、短债占比高
- 若缺少周期证据/负债结构数据：标注 `[未核验]` 并要求补齐。

### B. 回款/应收风险
- 应收账款增长显著快于收入（或应收/收入、应收周转天数显著恶化）→ 红旗
- 净利润为正但经营现金流为负（持续）→ 红旗（常见于回款压力/收入确认）

### C. 库存风险
- 存货显著上升 + 毛利率下行/价格下行 → 红旗（跌价与减值风险）
- 存货周转天数显著恶化 → 黄旗~红旗

### D. 价格敏感度/盈利弹性
- 若属于价格决定利润行业（大宗/化工/航运等）：
  - 单一产品价格/价差下行会导致利润快速收缩 → 默认黄旗；若叠加高杠杆/高库存 → 红旗

### E. 资本开支与现金流覆盖
- 资本开支（CAPEX）长期高于经营现金流/自由现金流 → 红旗（靠举债/融资续命风险）
- 在建工程/固定资产扩张明显但盈利与现金流未同步改善 → 黄旗~红旗

### F. 分红与自由现金流
- 高分红但自由现金流覆盖弱、或分红依赖举债/再融资 → 红旗（分红不可持续）
- 分红断断续续、且经营现金流波动大 → 黄旗（需核验股利政策）

### G. 资产质量（延续 + 强化）
- 商誉/净资产 > 30%（黄旗）；> 50%（红旗）
- 大额减值频繁出现（红旗：需看公告与附注）

### H. 杠杆与偿债（延续 + 强化）
- 资产负债率 > 70%（黄旗）；> 80%（红旗）
- 利息覆盖倍数过低（如 EBIT/利息 < 2，若可获得）→ 红旗
- 短债压力高（短债/货币资金过高，若可获得）→ 红旗

### I. 非经常性损益依赖（延续）
- 扣非净利润长期显著低于归母净利润（例如扣非/归母 < 0.7，连续 2 年）→ 红旗

### J. 治理与资本运作（延续）
- 控股股东质押比例 > 50%（红旗）
- 频繁再融资/并购且商誉快速上升（黄旗~红旗）

## 数据源（Data sources）
优先级：
1) 定期报告（年报/季报）+ 附注（最权威）
2) 巨潮资讯公告（分红、减值、诉讼、质押、问询函等）
3) 行情网站摘要（仅用于快速定位与交叉验证）

MVP 演示（离线/半离线）：
- 用户提供关键字段 CSV/JSON
- 或用 web_fetch 抓取公开页面摘要（会受反爬与结构变化影响）

## 最小可运行实现（MVP）

### 文件
- `demo_sanity_check.py`：读取 CSV，按规则输出红旗清单。

### CSV 输入字段（列名）
最低建议包含：
- code,name
- net_profit,net_profit_prev
- op_cf,op_cf_prev
- equity,goodwill
- total_assets,total_liab
- non_recurring_profit
- pledge_ratio（0~1）

建议新增（可选但强烈建议，用于扩展红旗）：
- revenue,revenue_prev
- ar（应收账款）,ar_prev
- inventory,inventory_prev
- capex（资本开支，若能提供）,fcf（自由现金流，若能提供）
- cash_dividend（现金分红）,cash_dividend_prev
- interest_expense,ebit（用于利息覆盖，若能提供）
- short_debt,cash（用于短债压力，若能提供）

### 运行示例
```bash
python3 ~/.openclaw/workspace/skill-catalog/project/a-share-sanity-check/demo_sanity_check.py --csv data/sample_sanity_input.csv
```
