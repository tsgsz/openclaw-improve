# boardgame-deconstruction（桌游拆解）

## 适用场景
- 桌游机制拆解（新游/经典款）
- 竞品对比（同品类：工放、牌驱、区域控制、社交推理等）
- 规则诊断（为什么不好玩/为什么上手难/为什么复玩低）
- 原型设计输入（从现有桌游提炼可迁移机制）

## 方法框架（融合公开方法）
1. **MDA 三层拆解**
   - Mechanics（规则与组件）
   - Dynamics（玩家行为与博弈过程）
   - Aesthetics（情绪体验与乐趣来源）
2. **核心循环（Core Loop）**
   - 每回合“获取资源→做决策→产生反馈→改变局势”的闭环
3. **决策质量四件套**
   - 可见信息密度 / 隐藏信息密度
   - 短期收益 vs 长期收益权衡
   - 互动强度（直接冲突/间接卡位）
   - 纠错空间（落后追赶、策略转向成本）
4. **系统约束检查**
   - 雪球效应（领先者滚大）
   - 王杀（Kingmaking）
   - 回合停滞（downtime）
   - 策略单一路径（最优解锁死）
5. **市场可理解性维度（参考 BGG 常见口径）**
   - 复杂度/重量（Weight）
   - 教学成本
   - 复玩性

## 输出模板（固定）
1) **一句话定位**
- 这款游戏本质在卖什么体验？

2) **机制拆解（MDA）**
- Mechanics：
- Dynamics：
- Aesthetics：

3) **核心循环**
- 循环步骤：
- 回合关键决策点：
- 主要互动发生位置：

4) **平衡与风险**
- 潜在失衡点：
- 雪球/王杀/停滞风险：
- 新手痛点：

5) **可抄模块（给设计）**
- 可迁移机制 A/B/C：
- 适合迁移到的品类：
- 改造建议（保守版/激进版）：

6) **结论分级**
- 可玩性：高/中/低
- 教学门槛：高/中/低
- 复玩潜力：高/中/低

## 使用指令示例
- “用 boardgame-deconstruction 拆解《卡坦岛》”
- “对比《璀璨宝石》和《七大奇迹》：核心循环+互动强度+复玩性”
- “拆解《诡镇奇谈》并给我三条可迁移到我们项目的机制”

## 质量标准
- 禁止只复述规则，必须给出“机制→行为→体验”的因果链
- 至少指出 2 个可验证的风险点与 2 个可迁移模块
- 对比任务必须给出结论（谁更强、强在哪、代价是什么）
