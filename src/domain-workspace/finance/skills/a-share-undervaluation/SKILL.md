# a-share-undervaluation（A股低估筛选 + 行业/周期/基本盘二次分层）

> 目标：在给定约束下，从 A 股中筛出“相对低估/高股息/破净”等候选清单，并在估值粗筛后进行 **行业&周期&基本盘** 二次打分/分层，输出可解释理由与主要风险点。
>
> 重要声明：本技能仅做信息筛选与风险提示，不构成投资建议。

---

## 何时使用
- 用户请求：
  - “找 10 只低估股票”
  - “找高股息、分红稳定的股票”
  - “找破净（PB<1）且财务别太差的”
  - “在某行业里找估值便宜的龙头/现金牛”

不适用：
- 需要实时逐笔/盘口交易决策
- 需要完整回测/组合优化（应转量化研究流程）

---

## 输入
**必填（至少一种）：**
1) `mode=screen` + 筛选条件（如果有全市场列表数据源）
2) `mode=list` + 股票代码列表（最小可运行模式，推荐）

**可选筛选条件（常用）**
- 估值：`pb_max`、`pe_ttm_max`、`pe_ttm_min`
- 股息：`dividend_yield_min`（%），`dividend_years_min`
- 质量：`roe_min`、`revenue_cagr_min`、`debt_ratio_max`
- 风险过滤：`exclude_st=true`、`exclude_suspended=true`、`exclude_loss_makers=true`
- 分散：`industry_cap_per_sw1=2`
- 流动性：`avg_turnover_min`（近 N 日平均成交额）

---

## 默认参数（用户不指定时）
- exclude_st: true
- exclude_suspended: true（若无法判定则提示）

估值粗筛（通用默认，行业可调）：
- pb_max: 1.5（低估宽松）/ 1.0（破净严格）
- pe_ttm_max: 15（非金融；金融/周期需单独解释）

股息（高股息场景默认）：
- dividend_yield_min: 4%
- dividend_years_min: 3（若拿不到历史则标 `[未核验]` 并降级置信度）

质量（通用粗筛）：
- roe_min: 8%（银行等可放宽并单独说明）
- debt_ratio_max: 70%（金融行业不适用，需改用资本充足/杠杆等口径）

## 输出（Telegram 友好）
- `数据更新时间（Asia/Shanghai）`
- `数据源`（至少 1 主 + 1 交叉验证或注明单源演示）
- `筛选条件`
- `结果`：按“估值粗筛 → 二次分层”输出
  - **A档（更稳健）/ B档（可跟踪）/ C档（高不确定）**
  - 每只股票必须包含：
    - 代码/简称
    - 关键指标（PB/PE/股息率/ROE/负债率等）
    - **行业一句话 / 周期一句话 / 基本盘一句话**（缺失必须标 `[未核验]`）
    - 风险点（1~3 条）
- `风险提示`（不构成投资建议）

---

## 方法论：两阶段筛选（必须）

### Stage 1：估值粗筛（快）
- 目的：用 PB/PE/股息率等把候选池缩小到可人工核验的规模。
- 注意：**低估≠便宜**，Stage 1 只回答“相对便宜”，不回答“能否穿越周期/是否有雷”。

### Stage 2：行业&周期&基本盘 二次打分/分层（防价值陷阱）
- 目的：在“便宜”的候选里，优先找 **解释得通、可持续、风险可控** 的。
- 形式：规则引擎/评分表（见 `rules_v1.md`）。

**二次分层输出规则（默认）：**
- 二次得分（0~100）：
  - **≥70：A档（更稳健）**
  - **50~69：B档（可跟踪）**
  - **<50：C档（高不确定/更像周期下注或基本盘存疑）**

**降级策略（数据缺失/无法核验时必须执行）：**
- 任一维度（行业/周期/基本盘）出现关键项无法核验：
  - 在该股条目中标注 `[未核验]`
  - 二次得分 **上限封顶为 69（不能进 A 档）**
- 如果基本盘关键现金流/负债结构完全缺失：
  - 二次得分 **上限封顶为 49（只能进 C 档）**

## 数据源（Data sources）
**主来源（默认）：**
- 东方财富 push2：`/api/qt/stock/get`（可快速抓取行情与部分估值字段）

**交叉验证（建议至少一个）：**
- 巨潮资讯（公告/分红方案/审计意见）
- 新浪/腾讯财经（估值与分红摘要校验）
- 行业/政策：公司年报（经营讨论与分析/风险章节）

注意：不同网站的 PE/PB/股息率口径可能不同（TTM/静态、更新时间点、是否剔除非经常性损益等），输出必须注明。

---

## 注意事项（Pitfalls）
- 低 PB 常见于：周期下行、盈利恶化、资产质量担忧、金融地产政策不确定等；必须提示“低估可能是价值陷阱”。
- 高股息率常常是“历史分红/当前股价”推导，未来分红不保证；需核验现金流覆盖。
- 金融（银行/券商/保险）与地产的指标体系不同：PE/PB可用但需结合资产质量、拨备、资本充足等；本技能默认只做“粗筛 + 二次分层 + 风险提示”。

## 最小可运行实现（MVP）

本技能提供一个**无需大依赖**的 Python 演示：
- 输入：股票代码列表（如 600519、000001）
- 过程：调用东方财富 push2 接口抓取 JSON（字段可能变动）；做简单阈值过滤；输出候选与原始字段快照
- 输出：Telegram 友好文本

### 文件
- `demo_em_fetch.py`：从 Eastmoney 拉取单只/多只股票的原始数据并做简单筛选（Stage 1）

### 运行示例
```bash
python3 ~/.openclaw/workspace/skill-catalog/project/a-share-undervaluation/demo_em_fetch.py --codes 600519,000001 --pb-max 1.5 --pe-max 15
```

> Stage 2（二次分层）默认以“web_fetch 抓年报/公告 + 规则引擎评分表”人工/半自动完成；如需自动化，可在后续迭代加解析器。
