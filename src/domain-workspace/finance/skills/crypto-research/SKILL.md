# 加密研究

专业加密货币研究方法论，包含基本面分析、链上分析、代币经济学、估值模型。

## 核心原则

1. **数据必须脚本验证**：所有价格、TVL、链上数据必须通过脚本获取，禁止估算
2. **交叉验证**：单一数据源不可靠，必须 2+ 来源确认
3. **风险优先**：先说风险，再说机会

## 研究框架

### 1. 基本面分析

#### 1.1 项目定位
- 解决什么问题？
- 目标市场多大？
- 竞品是谁？差异化在哪？

#### 1.2 团队背景
- 核心团队实名 + 可查历史
- 投资机构背景（a16z/Paradigm 等顶级机构加分）
- 开发者活跃度（GitHub 提交频率）

#### 1.3 技术实力
- 代码质量（是否有测试、文档）
- 技术创新性（非 Fork）
- 路线图执行历史

### 2. 代币经济学

#### 2.1 代币用途
- 治理权？
- 费用捕获？
- 质押收益？
- **关键：代币价值与协议收入是否挂钩**

#### 2.2 供应分析
```
必须获取的数据（通过脚本）：
- 当前流通量 (Circulating Supply)
- 最大供应量 (Max Supply)
- 完全稀释估值 (FDV)
- 解锁时间表 (TokenUnlocks)
- 通胀率 (Annual Inflation)

风险信号：
🔴 FDV > 10x MC（90%+ 代币未解锁）
🔴 团队/投资人占比 > 40%
🔴 30 天内有 > 5% 代币解锁
```

#### 2.3 需求驱动
- 协议收入是否增长？
- 代币有实际使用场景？
- 回购/销毁机制？

### 3. 链上分析

#### 3.1 持仓分布
```
必须获取的数据：
- Top 10 地址持仓占比
- 交易所持仓占比
- 合约锁仓量

风险信号：
🔴 Top 10 > 50%
🔴 单地址 > 20%
🔴 交易所持仓骤降（砸盘预警）
```

#### 3.2 资金流向
- 聪明钱（Smart Money）流向
- 大额转账监控
- 交易所净流入/流出

#### 3.3 TVL 分析
```
必须获取的数据：
- 当前 TVL（DeFiLlama）
- TVL 趋势（30/90/180 天）
- TVL 构成（是否单一资产主导）
- 真实收益（Real Yield）

关键指标：
- FDV/TVL 比率（越低越好，<3 合理）
- MC/TVL 比率（<1 可能被低估）
```

### 4. 合约安全

#### 4.1 审计情况
- 审计机构信誉（OpenZeppelin/Trail of Bits/CertiK）
- 审计时间（6 个月内）
- 漏洞修复状态

#### 4.2 权限检查
```
必须验证：
- 合约是否放弃所有权（Renounced）
- 是否可 mint 新币
- 是否可冻结转账
- 是否可升级（Proxy 风险）

风险信号：
🔴 团队仍拥有 mint 权限
🔴 可任意冻结用户资产
🔴 无时间锁（Timelock）
```

### 5. 估值模型

#### 5.1 相对估值
```
常用比率：
- FDV/TVL：对比同类协议
- P/S (Price/Sales)：协议收入倍数
- P/F (Price/Fees)：费用倍数
- MC/Revenue：市值收入比

对比标的：
- 同赛道头部（UNI vs SUSHI）
- 同类型（DEX vs DEX）
- 同阶段（早期 vs 成熟期）
```

#### 5.2 绝对估值
- DCF（现金流折现，适用有收入的协议）
- Metcalfe 定律（网络效应估值）
