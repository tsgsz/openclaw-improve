# A股深度核验

> A股深度核验Skill：用于初筛报告后，对"基本面红旗"与"分红/现金流可持续性"进行交叉核验与完成度追踪。
> 
> **免责声明**：本Skill仅用于信息整理与风险识别，不构成任何投资建议。所有标注`[未核验]`的字段需用户自行核实。

---

## 何时使用

在以下场景必须/建议使用本Skill：

| 场景 | 建议深度等级 | 说明 |
|------|-------------|------|
| 初筛报告涉及**分红承诺/高股息** | 标准/深度 | 核验分红可持续性 |
| **周期股**（能源/化工/钢铁/有色等） | 标准/深度 | 核验周期位置与财务弹性 |
| **高杠杆行业**（银行/保险/地产/基建） | 深度 | 核验负债结构与资产质量 |
| 发现**红旗信号**（高应收/高存货/高质押） | 深度 | 逐项核验风险敞口 |
| 常规初筛后质量抽检 | 快速 | 抽检核心指标一致性 |

**触发条件总结**：
- ✅ 涉及分红承诺/高股息策略 → **强制启用**
- ✅ 强周期行业（大宗/化工/钢铁/有色/航运） → **强制启用**
- ✅ 金融地产基建（高杠杆） → **强制启用**
- ✅ 红旗指标≥2项 → **建议深度**
- ⚠️ 其他情况 → **可选，快速抽检**

## 输入参数

```yaml
task:
  type: "deep_verification"
  inputs:
    stock_codes:          # 股票代码列表（必需）
      - "600000.SH"
      - "000001.SZ"
    initial_report_path:  "reports/screening_20240205.md"  # 初筛报告路径（可选）
    verification_level:   "standard"  # 核验深度：quick(快速)/standard(标准)/deep(深度)
    industry_hints:       # 行业提示（可选，用于选择核验模板）
      "600000": "banking"
      "000001": "banking"
  outputs:
    verification_patch:   "reports/verification_patch_20240205.json"
    completion_tracker:   "reports/verification_completion_20240205.md"
```

## 数据源优先级（严格）

**绝对禁止**：仅依赖百科（百度百科/维基百科）作为唯一来源

| 优先级 | 类型 | 示例 | 用途 |
|--------|------|------|------|
| 1 | **交易所官方公告** | 上交所/深交所/北交所公告、招股说明书 | 分红方案、重大事项、股权质押 |
| 2 | **年报/季报摘要** | 巨潮资讯、公司官网披露 | 财务数据、经营讨论、风险章节 |
| 3 | **权威财经媒体** | 证券时报、上海证券报、财新、第一财经 | 行业分析、政策解读、独家报道 |
| 4 | **财经数据平台** | 东方财富、Wind、同花顺、雪球 | 行情数据、财务摘要、估值指标 |
| 5 | **行业/协会数据** | 中钢协、中煤协、银保监会数据 | 行业景气、价格指数、政策口径 |
| ❌ | **百科类** | 百度百科、维基百科、互动百科 | **禁止作为独立来源，仅作背景参考** |

**交叉核验规则**：
- 关键财务数据（营收/利润/现金流）：至少2个来源
- 分红信息：以交易所公告为准，数据平台仅作参考
- 行业/周期判断：优先年报"经营讨论"，其次权威媒体

## 行业核验清单模板

### 银行业

**分红逻辑核验**：
- [ ] 分红率（分红/净利润）历史趋势
- [ ] 核心一级资本充足率是否达标（≥7.5%）
- [ ] 净利润增长是否支撑分红持续
- [ ] 是否收到监管分红限制指引

**现金流关注点**：
- [ ] 净息差（NIM）趋势
- [ ] 不良率与拨备覆盖率
- [ ] 存款结构（活期/定期比例）
- [ ] 同业负债依赖度

**红旗指标**：
- [ ] 关注类贷款占比上升
- [ ] 单一客户集中度超标
- [ ] 表外理财规模异常
- [ ] 房地产敞口占比

### 保险业

**分红逻辑核验**：
- [ ] 偿付能力充足率（综合/核心）
- [ ] 内含价值（EV）增长
- [ ] 新业务价值（NBV）趋势
- [ ] 股息来源于投资收益还是承保利润

**现金流关注点**：
- [ ] 保费收入质量（期缴/趸缴比例）
- [ ] 退保率变化
- [ ] 投资收益率与负债成本匹配
- [ ] 再保险安排

**红旗指标**：
- [ ] 偿付能力充足率接近监管红线
- [ ] 投资资产中高风险类别占比
- [ ] 重大赔付事件未决
- [ ] 关联交易规模

### 基建/公用事业

**分红逻辑核验**：
- [ ] 政府回款周期与应收账款
- [ ] 项目资本金到位情况
- [ ] 特许经营期限与续期风险
- [ ] 补贴依赖度

**现金流关注点**：
- [ ] PPP项目现金流回流
- [ ] 资本开支（CAPEX）规模与资金来源
- [ ] 应收账款周转天数
- [ ] 经营性现金流/净利润比值

**红旗指标**：
- [ ] 地方政府支付延迟/违约
- [ ] 项目停工/延期
- [ ] 高PPP项目占比+低现金流
- [ ] 有息负债快速扩张

### 周期行业（能源/化工/钢铁/有色）

**分红逻辑核验**：
- [ ] 分红与商品价格周期匹配度
- [ ] 资本开支计划与分红平衡
- [ ] 存货跌价准备计提
- [ ] 周期下行期分红历史

**现金流关注点**：
- [ ] 经营性现金流对商品价格敏感度
- [ ] 存货与预付账款变化
- [ ] 资本开支纪律性
- [ ] 套期保值覆盖度

**红旗指标**：
- [ ] 高库存+价格下跌
- [ ] 高杠杆+利息覆盖<2
- [ ] 关联交易销售占比高
- [ ] 资产减值计提不充分

### 电信/通信

**分红逻辑核验**：
- [ ] 资本开支（5G建设）与分红平衡
- [ ] 用户ARPU趋势
- [ ] 铁塔租赁成本
- [ ] 提速降费政策影响

**现金流关注点**：
- [ ] 自由现金流（经营现金流-资本开支）
- [ ] 预付费vs后付费用户结构
- [ ] 设备折旧政策
- [ ] 频谱使用费摊销

**红旗指标**：
- [ ] 用户流失加速
- [ ] 5G资本开支超预算
- [ ] 监管罚款或反垄断调查
- [ ] 重大技术迭代风险（如6G替代）

## 完成度追踪机制

### 核验项清单（每只共10项）

| # | 核验项 | 说明 | 数据源 |
|---|--------|------|--------|
| 1 | 分红历史 | 近5年分红连续性 | 交易所公告 |
| 2 | 分红覆盖 | 经营现金流/分红 | 年报/季报 |
| 3 | 现金流质量 | 经营现金流vs净利润 | 年报/季报 |
| 4 | 应收账款 | 应收增速vs营收增速 | 年报/季报 |
| 5 | 存货水平 | 存货周转天数趋势 | 年报/季报 |
| 6 | 负债结构 | 有息负债/总资产 | 年报/季报 |
| 7 | 短债压力 | 短债/现金比 | 年报/季报 |
| 8 | 关联交易 | 关联销售/采购占比 | 年报 |
| 9 | 股权质押 | 大股东质押比例 | 交易所公告 |
| 10 | 行业特项 | 按行业模板选择 | 多种 |

### 完成度状态标注

```
已核验 (x/10)：✅
待年报披露：📄
待公告：📢
待监管文件：📋
信息不可得：❓
```

### 完成度更新流程

1. **初次核验**：完成可获取项，标注待披露项
2. **定期更新**：季报/年报披露后自动补全
3. **触发更新**：重大公告（分红、并购、诉讼）后即时更新
4. **最终确认**：完成度≥80%时输出"核验通过"标记

## 使用示例

### 示例1：高股息策略核验

```bash
# 输入
stock_codes: ["600000", "601398", "601288", "600036"]
verification_level: "standard"
industry_hints: {"all": "banking"}

# 输出核验补丁
重点关注：分红可持续性 + 资本充足率 + 拨备覆盖率
交叉核验：交易所分红公告 vs 年报净利润 vs 资本充足率披露
```

### 示例2：周期股深度核验

```bash
# 输入
stock_codes: ["600028", "601857", "600019"]
verification_level: "deep"
industry_hints: {"600028": "energy", "601857": "energy", "600019": "steel"}

# 输出核验补丁
重点关注：商品价格敏感性 + 存货跌价 + 资本开支纪律
交叉核验：行业协会价格数据 vs 公司产量披露 vs 期货价格
```

### 示例3：质量抽检

```bash
# 输入
stock_codes: ["300001", "300002"]
verification_level: "quick"

# 输出核验补丁
快速抽检：核心财务数据一致性 + 重大红旗信号
交叉核验：东方财富 vs 同花顺 vs 交易所公告
```

## 限制与免责声明

1. **不提供投资建议**：本Skill仅输出核验状态与风险信号，不包含买入/卖出建议
2. **数据时效性**：财务数据存在滞后，需关注披露时间点
3. **不确定性标注**：所有`[未核验]`或`unknown`状态需用户自行判断
4. **来源局限性**：无法获取未公开信息，不涉及内幕信息分析
